Changelog

All notable changes to Vanduo Framework are documented here.

v1.1.5 February 15, 2026 Latest

Framework Changes

  • New: Quad-Spinner Loader component

    .vd-dynamic-loader, .vd-dynamic-loader-grid, .vd-dynamic-loader-text added to spinner.css. Skeleton card overlay variant via .vd-dynamic-loader-card.

  • Improvement: Ubuntu as default font

    Theme customizer DEFAULTS.FONT changed from jetbrains-mono to ubuntu.

  • Improvement: Per-section icons in search

    doc-search.js buildIndex() now extracts icons from section headings (DOM path) or custom data icon field. Search results prefer per-item icon over category fallback.

  • Fix: Search.create() auto-initialization

    Search.create() now calls .init() on the returned instance automatically.

Documentation Changes

  • New: Dynamic Web tab

    Third documentation tab with SPA-like lazy loading. Sections load on demand via sections.json registry. Client-side routing, scroll-spy, and skeleton+quad-spinner loading placeholder.

  • New: 24 additional section fragments

    All documentation and concepts sections extracted as lazy-loadable HTML fragments (39 total).

  • Improvement: Per-section icons in sidebar

    All three doc pages now show matching icons in sidebar navigation links.

  • Improvement: Sidebar text size increased

    .doc-nav-link bumped to 0.875rem, .doc-nav-section to 0.8125rem, with improved icon spacing.

  • Fix: Icon consistency

    Unified icons across sidebar, search results, and section body headings on all pages. Fixed 7 mismatched icons in sections.json.

  • Fix: Section insertion order

    Dynamic page now inserts loaded sections in registry order using insertBefore, preventing scroll-spy confusion.

v1.1.0 February 7, 2026 Breaking Changes

Breaking Changes

  • CSS Class Prefixing

    All CSS classes now use the vd- prefix to prevent collisions with other frameworks. Examples: .btn → .vd-btn, .container → .vd-container, .card → .vd-card. See MIGRATION.md for complete class mapping.

  • Manual Initialization Required

    Framework no longer auto-initializes on page load. You must explicitly call Vanduo.init() after loading the script. This allows better control over initialization timing in SPAs and complex applications.

New Features

  • Multiple Module Formats

    Added ESM (vanduo.esm.js) and CJS (vanduo.cjs.js) build outputs alongside IIFE format. Modern bundlers can now tree-shake unused components.

  • Package.json Exports Map

    Added proper exports field in package.json for clean imports: import { Vanduo } from 'vanduo-framework'.

Improvements

  • CSS Namespace Isolation

    The vd- prefix ensures Vanduo can be safely embedded in applications using Bootstrap, Tailwind, or other CSS frameworks without class name collisions.
