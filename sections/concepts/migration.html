Migration Guide

Upgrading from v1.0.x to v1.1.0 requires updating class names, CSS variables, and initialization code. This guide walks you through every step.

Estimated time: Most projects migrate in under 10 minutes. A migration script is available to automate the bulk of the work.

Step 1Update HTML Class Names

All framework classes now use the vd- prefix. Update every class reference in your HTML files.

Before (v1.0.x)

<div class="container">
  <div class="row">
    <div class="col-6">
      <div class="card">
        <button class="btn btn-primary">Click</button>
      </div>
    </div>
  </div>
</div>

After (v1.1.0)

<div class="vd-container">
  <div class="vd-row">
    <div class="vd-col-6">
      <div class="vd-card">
        <button class="vd-btn vd-btn-primary">Click</button>
      </div>
    </div>
  </div>
</div>

Step 2Update CSS Custom Properties

All CSS variables are now prefixed with --vd-. Update any custom styles that reference framework tokens.

Before (v1.0.x)

.my-component {
  color: var(--color-primary);
  background: var(--bg-primary);
  padding: var(--spacing-8);
  font-family: var(--font-family-base);
}

After (v1.1.0)

.my-component {
  color: var(--vd-color-primary);
  background: var(--vd-bg-primary);
  padding: var(--vd-spacing-8);
  font-family: var(--vd-font-family-base);
}

Step 3Update JavaScript Initialization

Vanduo no longer auto-initializes on page load. You must call Vanduo.init() manually.

Before (v1.0.x)

<!-- Auto-initialized on DOMContentLoaded -->
<script src="vanduo.js"></script>

<!-- No init call needed -->

After (v1.1.0)

<script src="vanduo.iife.min.js"></script>
<script>
  document.addEventListener('DOMContentLoaded', () => {
    Vanduo.init();
  });
</script>

Automated Migration Script

We've created a Node.js migration script to automate class name and variable updates across your project:

# Open the migration guide
https://github.com/Nostromo-618/vanduo-framework/blob/main/MIGRATION.md

# Run on your project (see guide for usage)
node migrate-v1.1.0.js ./my-project

# Review changes
git diff
Important: Always review the script's changes with git diff before committing. The script handles most cases but manual review ensures nothing is missed.

Migration Checklist

Use this checklist to make sure you haven't missed anything:

  • HTML classes — All .btn, .card, .container, etc. updated to .vd-btn, .vd-card, .vd-container
  • CSS variables — All --color-*, --bg-*, --font-* updated to --vd-color-*, --vd-bg-*, --vd-font-*
  • JS initialization — Vanduo.init() is called after DOM is ready
  • Module format — Script tag updated to new filename (vanduo.iife.min.js, vanduo.esm.min.js, or vanduo.cjs.min.js)
  • Cleanup calls — If using SPA routing, Vanduo.destroyAll() added before re-init
  • Browser tested — Verified layout and interactive components work as expected
