Breaking Changes

Breaking changes are modifications that require you to update your existing code when upgrading. Here is a complete summary of what changed in v1.1.0 and how to adapt.

Summary of Breaking Changes in v1.1.0

Change v1.0.x v1.1.0 Impact
CSS Classes .btn, .card .vd-btn, .vd-card High
CSS Variables --color-primary --vd-color-primary Medium
Initialization Auto-init on page load Manual Vanduo.init() High
Module Format IIFE only IIFE, ESM, CJS Low
Lifecycle No cleanup API Vanduo.destroyAll() Low

High ImpactCSS Class Namespacing

Every framework class is now prefixed with vd- to prevent conflicts with other libraries and your own styles.

v1.0.x

<button class="btn btn-primary">Save</button>
<div class="card">
  <div class="card-header">Title</div>
  <div class="card-body">Content</div>
</div>
<div class="alert alert-success">Done!</div>

v1.1.0

<button class="vd-btn vd-btn-primary">Save</button>
<div class="vd-card">
  <div class="vd-card-header">Title</div>
  <div class="vd-card-body">Content</div>
</div>
<div class="vd-alert vd-alert-success">Done!</div>

Medium ImpactCSS Variable Prefixing

All CSS custom properties are now prefixed with --vd-. This affects any custom CSS that references framework tokens.

v1.0.x

:root {
  --color-primary: #6366f1;
}

.custom {
  color: var(--color-primary);
  background: var(--bg-secondary);
  box-shadow: var(--shadow-md);
}

v1.1.0

:root {
  --vd-color-primary: #6366f1;
}

.custom {
  color: var(--vd-color-primary);
  background: var(--vd-bg-secondary);
  box-shadow: var(--vd-shadow-md);
}

High ImpactManual Initialization

Vanduo no longer auto-initializes. This gives you full control over when components are activated — especially important in SPAs.

v1.0.x

<!-- Just include the script -->
<script src="vanduo.js"></script>

<!-- Components auto-initialized
     on DOMContentLoaded -->

v1.1.0

<script src="vanduo.iife.min.js"></script>
<script>
  document.addEventListener('DOMContentLoaded', () => {
    Vanduo.init();
  });
</script>
Tip: Use the automated migration script to handle most of these changes automatically. See the Migration Guide for step-by-step instructions and the Node.js script.
