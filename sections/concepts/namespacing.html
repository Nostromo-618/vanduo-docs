CSS Namespacing Strategy

One of the most important architectural decisions in Vanduo v1.1.0 is the introduction of comprehensive CSS namespacing. Every framework class is prefixed with vd- to prevent collisions and ensure safe integration.

Before v1.1.0

/* Generic class names */
.container { }
.row { }
.col-6 { }
.btn { }
.card { }
.alert { }

/* Problems:
   - Conflicts with Bootstrap
   - Conflicts with Tailwind
   - Conflicts with your own .btn
   - Unpredictable in large apps
*/

Since v1.1.0

/* Namespaced class names */
.vd-container { }
.vd-row { }
.vd-col-6 { }
.vd-btn { }
.vd-card { }
.vd-alert { }

/* Benefits:
   - Zero conflicts guaranteed
   - Clear framework origin
   - Safe embedding in any app
   - Multiple framework friendly
*/

Why This Matters

Note: the "Before v1.1.0" snippet above uses legacy, unprefixed class names for comparison only.

Without Namespacing

  • Cannot safely embed in WordPress themes
  • Conflicts with Bootstrap-based admin panels
  • Cannot use alongside Tailwind projects
  • Your .btn styles may override ours
  • Difficult to isolate in micro-frontends

With vd- Prefix

  • Safe embedding in any environment
  • Works alongside Bootstrap, Tailwind, etc.
  • Your .btn remains untouched
  • Shadow DOM friendly
  • Perfect for widget/embed scenarios
