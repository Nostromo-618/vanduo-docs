Getting Started from Zero Guide

Welcome! This guide walks you through building your very first web page with Vanduo — from an empty folder to a working, styled site. No prior framework experience needed. Think of this as a senior developer sitting next to you, walking you through every step.

What You'll Build

By the end of this guide, you'll have a complete page with:

  • A responsive navbar with logo and links
  • A hero section with text and a call-to-action button
  • A card grid showing content in columns
  • A footer at the bottom
  • Dark/light theme toggle built in

All of this in about 60 lines of HTML — no CSS to write, no JavaScript to configure.

Step 1 of 7 Create Your Project Folder

Open your terminal and create a new folder for your project. This is where all your files will live:

mkdir my-first-vanduo-site
cd my-first-vanduo-site

Now create an index.html file. You can use any text editor — VS Code, Sublime Text, or even Notepad:

touch index.html
Pro tip: If you're using VS Code, you can open the folder directly with code . after creating it. VS Code's Live Server extension is great for auto-refreshing as you edit!

Step 2 of 7 Add Vanduo to Your Page

The fastest way to get started is via CDN — no downloads, no installs, just two lines in your HTML. Open index.html and paste this starter template:

<!DOCTYPE html>
<html lang="en" data-theme="light">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Vanduo Site</title>

    <!-- Vanduo CSS — this gives you the entire design system -->
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/Nostromo-618/vanduo-framework@main/dist/vanduo.min.css">
</head>
<body>

    <!-- Your content will go here -->

    <!-- Vanduo JS — enables interactive components -->
    <script src="https://cdn.jsdelivr.net/gh/Nostromo-618/vanduo-framework@main/dist/vanduo.min.js"></script>
    <script>Vanduo.init();</script>
</body>
</html>
Pro tip: For production, pin to a specific version like @v1.1.6 instead of @main so your site doesn't unexpectedly change when the framework updates.
Why CDN? CDN (Content Delivery Network) hosts the files for you on fast servers worldwide. Your visitors download Vanduo from the nearest server — no setup on your end. Perfect for learning and prototyping.

Step 3 of 7 Add a Navbar

Replace the <!-- Your content will go here --> comment with a navbar. This gives you a responsive navigation bar that collapses on mobile:

<nav class="vd-navbar">
    <div class="vd-container">
        <a href="#" class="vd-navbar-brand">My Site</a>
        <button class="vd-navbar-toggler" data-toggle="collapse" data-target="#mainNav"
                aria-label="Toggle navigation">
            <i class="ph ph-list"></i>
        </button>
        <div id="mainNav" class="vd-navbar-collapse">
            <ul class="vd-navbar-nav">
                <li><a href="#" class="vd-nav-link">Home</a></li>
                <li><a href="#" class="vd-nav-link">About</a></li>
                <li><a href="#" class="vd-nav-link">Contact</a></li>
            </ul>
        </div>
    </div>
</nav>

What's happening here:

  • vd-navbar — The main navbar wrapper, handles styling and layout
  • vd-navbar-brand — Your site name or logo
  • vd-navbar-toggler — The hamburger button that appears on mobile
  • data-toggle="collapse" — Tells Vanduo's JS to make this a collapsible element
  • vd-navbar-nav — The list of navigation links

Step 4 of 7 Add a Hero Section

Below the </nav> tag, add a hero section — the big, eye-catching area at the top of your page:

<section class="vd-container vd-py-13">
    <div class="vd-row">
        <div class="vd-col-12 vd-col-md-8">
            <h1>Welcome to My Site</h1>
            <p class="vd-text-lg vd-text-muted vd-mb-8">
                This is my first website built with the Vanduo framework.
                It looks great and I didn't write a single line of CSS!
            </p>
            <a href="#" class="vd-btn vd-btn-primary vd-btn-lg">
                <i class="ph ph-rocket-launch"></i> Get Started
            </a>
        </div>
    </div>
</section>
Why vd-col-md-8? This means "take up 8 out of 12 columns on medium screens and above." On mobile, it stretches full width (vd-col-12). This keeps your text readable — nobody likes lines that stretch across the entire screen!

Step 5 of 7 Add Content Cards

Below the hero, let's add a grid of cards — a common pattern for showcasing features, blog posts, or services:

<section class="vd-container vd-py-8">
    <h2 class="vd-mb-8">Features</h2>
    <div class="vd-row">
        <div class="vd-col-12 vd-col-md-4 vd-mb-5">
            <div class="vd-card">
                <div class="vd-card-body">
                    <h3><i class="ph ph-lightning"></i> Fast</h3>
                    <p>Lightweight CSS with no heavy dependencies.</p>
                </div>
            </div>
        </div>
        <div class="vd-col-12 vd-col-md-4 vd-mb-5">
            <div class="vd-card">
                <div class="vd-card-body">
                    <h3><i class="ph ph-devices"></i> Responsive</h3>
                    <p>Looks great on phones, tablets, and desktops.</p>
                </div>
            </div>
        </div>
        <div class="vd-col-12 vd-col-md-4 vd-mb-5">
            <div class="vd-card">
                <div class="vd-card-body">
                    <h3><i class="ph ph-moon"></i> Dark Mode</h3>
                    <p>Built-in light and dark theme support.</p>
                </div>
            </div>
        </div>
    </div>
</section>

The grid in action: Each card uses vd-col-md-4 (4 out of 12 = one-third width), so you get 3 columns on desktop. On mobile they stack vertically because of vd-col-12.

Step 6 of 7 Add a Footer

Every site needs a footer. Add this before the closing </body> tag (but above the <script> tags):

<footer class="vd-footer">
    <div class="vd-container">
        <div class="vd-row">
            <div class="vd-col-12 vd-col-md-6">
                <p>&copy; 2026 My Site. Built with Vanduo.</p>
            </div>
            <div class="vd-col-12 vd-col-md-6" style="text-align: right;">
                <a href="#">Privacy</a> &middot;
                <a href="#">Terms</a>
            </div>
        </div>
    </div>
</footer>

Step 7 of 7 Open in Your Browser

Save your index.html and open it in a web browser. You can simply double-click the file, or use a local server:

# Option A: Just double-click index.html in your file manager

# Option B: Use a simple HTTP server (recommended)
npx http-server . -p 3000
# Then open http://localhost:3000
If you see a styled page with a navbar, hero, cards, and footer — you did it! You've built your first Vanduo site. The navbar collapses on mobile, the cards stack responsively, and everything has consistent typography and spacing.

Bonus: Add a Theme Toggle

Want dark mode? Just add a button anywhere in your page (the navbar is a great spot):

<button data-toggle="theme" aria-label="Toggle theme">
    <i class="ph ph-moon"></i>
</button>

That's it — one attribute. Vanduo's JS handles the rest: toggling data-theme on the <html> element, saving the preference to localStorage, and updating all colors automatically.

Why does this work? Vanduo uses CSS custom properties (variables) that change based on the data-theme attribute. When you switch themes, every color in the system updates — buttons, cards, text, backgrounds — all from a single attribute change.
