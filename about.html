About

Mission, vision, and constraints that shape Vanduo Framework

Mission

To give every developer a framework that feels like writing plain HTML again -- simple enough to start in minutes, powerful enough to ship real products. No abstractions to learn, no toolchains to configure, just water-clear code.

Vision

A web where lightweight sites are the norm, not the exception. Where choosing simplicity is never a compromise on beauty, accessibility, or performance.

Constraints

Zero dependencies. No build step. No npm install. Every kilobyte earns its place. Vanduo ships as pure HTML, CSS, and JavaScript -- if a feature can't justify its weight, it doesn't get in. These limits aren't restrictions; they're the reason the framework stays fast, portable, and honest.

Perspective

We picture a development landscape where reaching for a massive framework is a choice, not a reflex. Vanduo exists so that static sites, landing pages, and documentation can look polished without pulling in hundreds of packages.

Discipline

No transpilers, no bundlers, no runtime surprises. The entire framework fits in a single stylesheet and one ES module. Browser-native features first; polyfills never.

Essence

Strip away everything that doesn't serve the developer or the end user. What remains is Vanduo -- essential, like water.

Support Vanduo Development

Help keep the framework lean, accessible, and thoughtfully designed. Your support funds maintenance, documentation, and new features.
